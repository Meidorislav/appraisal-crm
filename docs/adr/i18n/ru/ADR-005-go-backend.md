# ADR-005: Go для бэкенд-сервисов

> Русская версия · [English version](../../ADR-005-go-backend.md)

**Статус:** На рассмотрении  
**Дата:** 2026-06-15

## Контекст

Бэкенд состоит из нескольких независимых сервисов (API Gateway, Request,
Inspect, Review, Notification). Нужны язык и среда исполнения, которые хорошо
работают при конкурентной нагрузке, компилируются в единый бинарник для
простого деплоя и удобны в долгосрочной поддержке.

## Решение

Использовать Go для всех бэкенд-сервисов.

## Последствия

**Положительные:**
- Отличная модель конкурентности (горутины) — хорошо подходит для API Gateway
  и event-driven Notification Service
- Компилируется в единый статический бинарник — простые Docker-образы, нет
  зависимостей от рантайма
- Мощная стандартная библиотека покрывает HTTP, JSON и работу с БД без тяжёлых
  фреймворков
- Быстрая компиляция поддерживает быструю итерацию

**Отрицательные:**
- Менее выразителен, чем языки высокого уровня, для сложной доменной логики
- Экосистема уже, чем у Java/Node.js, для некоторых интеграций (например,
  клиентские библиотеки Keycloak Admin)
- Команда должна хорошо знать Go; онбординг новых разработчиков может занять
  время
