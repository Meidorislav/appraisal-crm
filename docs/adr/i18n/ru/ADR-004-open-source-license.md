# ADR-004: Открытая лицензия (Apache 2.0)

> Русская версия · [English version](../../ADR-004-open-source-license.md)

**Статус:** На рассмотрении  
**Дата:** 2026-06-15

## Контекст

Клиент требует, чтобы исходный код оставался публично доступным после передачи.
Лицензия должна быть достаточно permissive, чтобы клиент мог свободно
использовать, изменять и распространять ПО, и при этом совместима с
open-source зависимостями проекта (экосистема Go, React и др.).

Apache 2.0 выбрана вместо MIT, поскольку это корпоративный проект: Apache 2.0
включает явный патентный грант — контрибьюторы не могут впоследствии предъявлять
патентные претензии пользователям программного обеспечения. MIT такой защиты не
обеспечивает.

## Решение

Выпустить весь исходный код под лицензией **Apache 2.0**.

## Последствия

**Положительные:**
- Явный патентный грант защищает пользователей и клиента от патентных претензий
  со стороны контрибьюторов
- Клиент имеет полную свободу использовать, изменять и распространять код
- Совместима со всеми основными open-source зависимостями в стеке
- Привлекает потенциальных контрибьюторов из сообщества

**Отрицательные:**
- Любой может форкнуть и использовать код без обратного вклада
- Нет коммерческой защиты для команды разработки
- Требует сохранения NOTICE-файла с атрибуцией в производных работах
- Открытый код может помочь злоумышленникам быстрее обнаружить уязвимости

**Митигация риска безопасности:**
- Секреты (API-ключи, пароли, токены) никогда не хранятся в репозитории —
  только в переменных окружения или secrets vault
- Файлы `.env` внесены в `.gitignore`
- SAST (например, `gosec` для Go) запускается в CI для обнаружения типичных
  уязвимостей до попадания кода в основную ветку
- Детали инфраструктуры (IP серверов, реальные схемы БД) не коммитятся
