# ADR-003: База данных на сервис

> Русская версия · [English version](../../ADR-003-database-per-service.md)

**Статус:** На рассмотрении  
**Дата:** 2026-06-15

## Контекст

Система построена на микросервисной архитектуре. Общая база данных между
сервисами создаёт тесную связанность — изменения схемы в одном сервисе могут
сломать другие, а сервисы не могут деплоиться и масштабироваться независимо.

## Решение

Каждый бэкенд-сервис владеет своей базой данных PostgreSQL:

| Сервис | База данных |
|--------|------------|
| Request Service | Request DB — заявки, статусы, данные клиентов |
| Inspect Service | Inspect DB — задания и данные осмотров |
| Review Service | Review DB — данные оценки и отчёты |

Сервисы взаимодействуют через API-вызовы (через API Gateway) или события Kafka,
но никогда не обращаются напрямую к базе данных другого сервиса.

## Последствия

**Положительные:**
- Сервисы деплоятся и масштабируются независимо
- Изменения схемы в одном сервисе не затрагивают другие
- Чёткое владение данными по доменам

**Отрицательные:**
- Запросы между сервисами требуют API-вызовов или event-driven репликации данных
- Больше баз данных для подготовки и резервного копирования
- Нет внешних ключей между границами сервисов
