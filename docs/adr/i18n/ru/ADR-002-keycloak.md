# ADR-002: Keycloak для аутентификации и авторизации

> Русская версия · [English version](../../ADR-002-keycloak.md)

**Статус:** На рассмотрении  
**Дата:** 2026-06-15

## Контекст

Система имеет четыре роли (Клиент, Оценщик, Осмотрщик, Администратор) с разными
уровнями доступа. Логика аутентификации и авторизации должна быть единой для
всех SPA и бэкенд-сервисов. Разработка собственной системы аутентификации
создаёт риски безопасности и значительные затраты на разработку.

## Решение

Использовать Keycloak в качестве identity provider. Keycloak обрабатывает потоки
OAuth2/OIDC, выдаёт JWT и управляет назначением ролей. API Gateway валидирует
токены при каждом запросе перед проксированием к бэкенд-сервисам.

## Последствия

**Положительные:**
- Никакого кастомного кода авторизации — снижает поверхность атаки
- Управление ролями централизовано в Keycloak Admin UI
- Стандартный OAuth2/OIDC — все SPA используют один и тот же flow
- Аккаунты клиентов можно создавать программно через Keycloak Admin API
  (BR-003: автоматическое создание аккаунта при подтверждении заявки)

**Отрицательные:**
- Добавляет ещё один сервис для деплоя и поддержки
- Keycloak ресурсоёмок для небольшого деплоя
- Команда должна быть знакома с конфигурацией Keycloak
