# ADR-006: S3 Yandex Cloud для хранения файлов

> Русская версия · [English version](../../ADR-006-s3-storage.md)

**Статус:** На рассмотрении  
**Дата:** 2026-06-15

## Контекст

Система хранит два типа файлов: фотографии с осмотра (загружаются осмотрщиками
в поле) и сформированные отчёты (PDF). Файлы должны храниться надёжно, быть
доступны через интернет и скачиваться клиентами без маршрутизации через бэкенд
при каждом запросе.

Срок хранения данных — **5 лет**.

## Решение

Использовать S3 Yandex Cloud для хранения всех файлов. Файлы загружаются
бэкенд-сервисами (Inspect Service — фото, Review Service — отчёты) и
предоставляются клиентам через presigned URL с коротким TTL — клиенты скачивают
напрямую из S3, минуя бэкенд.

## Последствия

**Положительные:**
- Клиенты скачивают файлы напрямую из S3 — нет нагрузки по пропускной
  способности на бэкенд-сервисы
- Presigned URL ограничивают доступ без необходимости аутентификации при
  каждом скачивании
- Управляемый сервис — нет инфраструктуры для поддержки хранилища
- S3-совместимый API — при необходимости можно заменить на любого
  S3-совместимого провайдера

**Отрицательные:**
- Постоянные расходы (ежемесячно/ежегодно) — зависят от общего объёма хранения
  за 5 лет
- Данные хранятся вне собственной инфраструктуры компании
- Требует доступа к интернету у осмотрщиков для загрузки фото в поле
